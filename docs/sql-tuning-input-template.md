# SQL 调优输入模板

将下面代码块中的内容完整复制到助手输入框，删除所有未填写的占位行后提交。数据库方言仍以输入框右下角的选择为准。

`SQL` 是唯一的系统必填项。要获得可验证的索引或改写建议，应同时提供 `现有索引`、`表结构`、文本 `执行计划`、表统计、OceanBase 版本和业务语义约束；缺失时助手仍会分析，但会降低结论确定性。`根因`、`优化建议`等历史结论不要填入模板，它们会干扰独立判断。

```text
数据库方言: OceanBase Oracle / OceanBase MySQL
数据库版本: OceanBase 4.x.x.x

SQL ID: <可选>

SQL:
<只粘贴一条完整 SELECT / INSERT / UPDATE / DELETE；保留绑定变量 ?；末尾分号可保留>

运行指标:
执行次数: <例如 21>
平均耗时: <例如 2008ms>
平均返回行数: <例如 1 行；DML 可填写平均影响行数>
逻辑读: <例如 2400000>
物理读: <例如 18300>
CPU占比: <例如 41.7%>
限流值: <可选>

表统计:
<表名>: <总行数> 行
<主过滤条件实际命中>: <行数> 行（<占总表比例>%）
<参与索引、排序或关联的列>: NDV=<基数>，NULL=<比例或行数>，热点值=<可选>
<分区信息 / 分区命中情况；无分区则填“无”>

表结构:
<粘贴本 SQL 涉及的所有表的完整 CREATE TABLE DDL，不要只粘贴列名>

现有索引:
<粘贴所有 PRIMARY KEY、UNIQUE、CREATE INDEX 定义；确认没有时必须明确填写“无”>

执行计划:
<直接粘贴文本 EXPLAIN / EXPLAIN ANALYZE 输出。不要只粘贴 Word 中的图标、截图标题或既有结论。>

业务语义约束:
- 运行环境 / 租户 / 数据库: <可选，例如 fqydhdb / JYZX1>
- 采集时间: <可选，YYYY-MM-DD HH:mm:ss>
- <哪些筛选、JOIN、排序、分页、NULL 语义不能改>
- <DML 允许更新哪些列，是否必须保持幂等、事务边界和影响行数>
- <默认值、去重规则、时间边界、结果排序等不可改变的要求>

允许建议类型: 诊断、索引、改写、验证
```

填写规则：

- `现有索引: 无` 是有效证据，不要留空。
- 将执行计划以纯文本粘贴；计划截图可另外作为图片上传，但不能替代文本计划。
- `允许建议类型` 只可填写 `诊断`、`索引`、`改写`、`验证`，可填其中一部分；填写 `全部` 等同于四项全开。
- 不要把历史系统给出的“根因”或“优化建议”混进运行指标。它们不是事实，会被当成待核验背景。
- 单次粘贴不超过 128 KiB；完整表 DDL、索引定义和计划文本优先保留，冗长日志可截取与该 SQL 有关的部分。
